# This document shows the response steps taken via Defender for Cloud.
## Steps:
- Defender for Key Vault set off an alarm which is visible in Defender for Cloud. The alert showed exactly what happened - the sign-in event happened through a TOR exit node. It is typically used by adversaries to hide their actual location - when I checked the alert it showed a city in France as source location, although the adversary (Me) was actually located in Spain.<img width="584" height="841" alt="alert" src="https://github.com/user-attachments/assets/6be1cfeb-a189-4063-93fa-1bba2188b30e" />
- I checked the recommended actions to prevent future attacks in Defender for Cloud. It helped me understand that the recommended network security setting for an Azure Key Vault is to either use Azure Private Link, or to completely disable public network access, allowing only trusted Microsoft services to bypass the firewall.
<img width="823" height="339" alt="prevent_future_attacks" src="https://github.com/user-attachments/assets/f6236d60-1d50-48bd-bfc3-4fcf92b62820" />
- Lastly, I went to the Key Vault and realized it was open for public access. I disabled the public access to prevent similar suspicious scenarios for the future.<img width="1523" height="847" alt="disable_public_access" src="https://github.com/user-attachments/assets/d9457b0d-7adf-49dd-b7e4-d8fe630adda5" />
